# 19-5 Changes

## Main Audit / Product Changes

- Teacher attendance now only allows marking attendance for the class where teacher is class teacher; teacher class cards show `My class` and hide invalid mark-attendance actions.
- WhatsApp logs now have proper status filters, retry for failed messages, full-message view, credits tile, and richer seeded logs.
- Teacher edit page now loads teacher by ID directly, fixing the `limit must be <= 100` validation error.
- Login redirects now preserve school ID in URL for principal, teacher, accountant, and legacy/root schools.
- New onboarded schools now wait for super admin approval before access; super admin can grant, revoke, and extend access; trial access auto-revokes after 30 days.
- Super admin panel now shows schools scheduled for deletion and pending approval clearly.
- Onboard page now only offers the free trial flow; removed Rs. 20,000 purchase option.
- Students page no longer shows fake fallback students for new schools.
- Student import now opens an instructional CSV modal before selecting a file.
- Student bulk selection now supports selecting all students across pagination and keeps correct selected count across pages.
- Homework submission tracking now has segmented status controls, autosave on blur, row-specific notes, class-average marks, and progress tracking.
- Attendance class selector now has search, Primary/Middle/Secondary grouping, and remembers last selected class per teacher.
- Homework create assignment now requires subject, removes `General` fallback, supports multiple attachments, rubric/answer-key files, estimated time, larger description, and basic formatting helpers.
- Homework assignment list now shows overdue age/severity, filters for Overdue/Due soon/Closed, and can nudge non-submitters on WhatsApp.

## Verification Used

- Backend type checks with `npm run lint --prefix backend`.
- Frontend production builds with `npm run build --prefix frontend`.
